## Overview

### What is SEO-CRO Balance?

The intersection between Search Engine Optimization (SEO) and Conversion Rate Optimization (CRO) represents one of the most critical yet challenging aspects of digital marketing. While both aim to improve business outcomes, they often pull in opposite directions.

**SEO Focus**: Traffic volume, rankings, visibility, crawlability
**CRO Focus**: Conversion quality, user action, revenue per visitor, friction reduction

**The Core Challenge**: SEO optimizations can hurt conversions, and CRO improvements can damage rankings.

## A/B Testing for SEO

### The Challenge of SEO A/B Testing

Unlike traditional CRO testing, SEO testing requires special considerations:
- Search engines must see consistent content
- Results take weeks/months to materialize
- Cloaking violations risk penalties
- Statistical significance is harder to achieve

### SEO-Safe Testing Methodologies

#### Method 1: **Time-Based Serial Testing**

Best for: Small to medium sites with limited templates

```
Process:
Week 1-4:  Baseline measurement (Variant A)
Week 5-8:  Implement Variant B
Week 9-12: Measure impact
Week 13+:  Keep winner, iterate

Controls:
- Same seasonality period year-over-year
- Account for algorithm updates
- Use control pages for comparison
```

**Real-World Example**: Blog Post Optimization

```
Test: Title Tag Optimization for Click-Through Rate

Site: Marketing blog with 200 posts
Target: Posts ranking positions 3-10

Variant A (Original):
"10 Email Marketing Tips for Small Businesses"
Average CTR: 3.2%
Rankings: Position 6

Variant B (Modified):
"Email Marketing Tips for Small Businesses [2024 Guide]"
Added: Current year, power word
Average CTR: 5.1% (+59%)
Rankings: Position 6 (stable)

Implementation:
```

```python
# Python script for title optimization testing
import pandas as pd
from datetime import datetime, timedelta

class SEOABTest:
    def __init__(self, test_duration_weeks=8):
        self.test_duration = test_duration_weeks
        self.baseline_data = []
        self.variant_data = []
    
    def collect_baseline(self, gsc_data):
        """Collect baseline performance from GSC"""
        return {
            'avg_position': gsc_data['position'].mean(),
            'avg_ctr': gsc_data['ctr'].mean(),
            'avg_clicks': gsc_data['clicks'].mean(),
            'impressions': gsc_data['impressions'].sum()
        }
    
    def apply_variant(self, pages, new_title_template):
        """Apply variant to test pages"""
        for page in pages:
            page.update_title(new_title_template.format(
                keyword=page.target_keyword,
                year=datetime.now().year
            ))
    
    def calculate_significance(self, baseline, variant):
        """Calculate statistical significance"""
        from scipy import stats
        
        # Chi-square test for CTR
        clicks_a = baseline['avg_clicks'] * self.test_duration * 7
        impr_a = baseline['impressions']
        clicks_b = variant['avg_clicks'] * self.test_duration * 7
        impr_b = variant['impressions']
        
        chi2, p_value = stats.chi2_contingency([
            [clicks_a, impr_a - clicks_a],
            [clicks_b, impr_b - clicks_b]
        ])[:2]
        
        return {
            'significant': p_value < 0.05,
            'p_value': p_value,
            'confidence': (1 - p_value) * 100
        }

# Usage Example
test = SEOABTest(test_duration_weeks=8)

# Collect baseline data
baseline_metrics = test.collect_baseline(gsc_baseline_data)

# Apply variant
test.apply_variant(target_pages, "{{keyword}} [2024 Guide]")

# Wait 8 weeks...

# Measure results
variant_metrics = test.collect_baseline(gsc_variant_data)

# Calculate significance
results = test.calculate_significance(baseline_metrics, variant_metrics)
print(f"Test is significant: {results['significant']}")
print(f"Confidence level: {results['confidence']:.1f}%")
```

#### Method 2: **Bucketed Template Testing**

Best for: Large sites with many similar pages

```
Strategy: Split similar pages into control/variant groups

Example: E-commerce Category Pages
Total pages: 500 category pages
Split: 250 control + 250 variant

Variant Changes:
- H1 structure modification
- Internal linking pattern
- Schema markup addition

Measurement:
- Organic traffic change (variant vs control)
- Rankings change
- Conversion rate impact
```

**Real Implementation**: E-commerce Internal Linking Test

```python
# Template A/B Testing Framework
class TemplateSEOTest:
    def __init__(self, site_pages):
        self.pages = site_pages
        self.control_group = []
        self.variant_group = []
    
    def create_matched_buckets(self):
        """Create statistically similar buckets"""
        from sklearn.cluster import KMeans
        import numpy as np
        
        # Features for matching
        features = pd.DataFrame({
            'current_traffic': [p.monthly_traffic for p in self.pages],
            'avg_position': [p.avg_ranking for p in self.pages],
            'page_authority': [p.domain_authority for p in self.pages],
            'word_count': [p.content_length for p in self.pages]
        })
        
        # Normalize features
        from sklearn.preprocessing import StandardScaler
        scaler = StandardScaler()
        features_scaled = scaler.fit_transform(features)
        
        # Create 2 clusters
        kmeans = KMeans(n_clusters=2, random_state=42)
        features['bucket'] = kmeans.fit_predict(features_scaled)
        
        # Assign buckets
        self.control_group = [self.pages[i] for i in 
                            features[features['bucket']==0].index]
        self.variant_group = [self.pages[i] for i in 
                             features[features['bucket']==1].index]
        
        return self.verify_bucket_similarity()
    
    def verify_bucket_similarity(self):
        """Verify buckets are statistically similar"""
        from scipy import stats
        
        control_traffic = [p.monthly_traffic for p in self.control_group]
        variant_traffic = [p.monthly_traffic for p in self.variant_group]
        
        # T-test to verify no significant difference
        t_stat, p_value = stats.ttest_ind(control_traffic, variant_traffic)
        
        return {
            'similar': p_value > 0.05,
            'p_value': p_value,
            'control_mean': np.mean(control_traffic),
            'variant_mean': np.mean(variant_traffic)
        }
    
    def apply_template_change(self, variant_template):
        """Apply changes only to variant group"""
        for page in self.variant_group:
            page.apply_template(variant_template)
    
    def measure_impact(self, weeks=12):
        """Measure differential impact"""
        control_delta = self.get_traffic_change(self.control_group)
        variant_delta = self.get_traffic_change(self.variant_group)
        
        # Calculate incremental lift
        lift = ((variant_delta - control_delta) / control_delta) * 100
        
        # Statistical significance test
        from scipy import stats
        control_data = [p.traffic_change for p in self.control_group]
        variant_data = [p.traffic_change for p in self.variant_group]
        t_stat, p_value = stats.ttest_ind(variant_data, control_data)
        
        return {
            'control_change': f"{control_delta:+.1f}%",
            'variant_change': f"{variant_delta:+.1f}%",
            'incremental_lift': f"{lift:+.1f}%",
            'significant': p_value < 0.05,
            'confidence': (1 - p_value) * 100
        }
    
    def get_traffic_change(self, page_group):
        """Calculate average traffic change for page group"""
        before = sum(p.traffic_before for p in page_group)
        after = sum(p.traffic_after for p in page_group)
        return ((after - before) / before) * 100

# Usage Example
pages = load_all_category_pages()  # Load your pages
test = TemplateSEOTest(pages)

# Create matched buckets
similarity = test.create_matched_buckets()
print(f"Buckets are similar: {similarity['similar']}")

# Apply variant (e.g., enhanced internal linking)
variant_template = {
    'related_products': 8,
    'category_links': 12,
    'semantic_linking': True
}
test.apply_template_change(variant_template)

# Wait 12 weeks for data...

# Measure impact
results = test.measure_impact(weeks=12)
print(f"Incremental lift: {results['incremental_lift']}")
print(f"Statistically significant: {results['significant']}")
```

#### Method 3: **Subdomain/Subfolder Split Testing**

Best for: Major architectural changes

```

- Complete redesign testing
- New site architecture
- Major UX changes
- Template overhauls

- Use consistent internal linking structure
```

```python
# DIY SEO Testing Framework
import google.auth
from googleapiclient.discovery import build
from google.oauth2.credentials import Credentials
import pandas as pd
from datetime import datetime, timedelta
import numpy as np

class GSCSEOTester:
    def __init__(self, property_url, credentials_path):
        self.property_url = property_url
        self.service = self.authenticate_gsc(credentials_path)
    
    def authenticate_gsc(self, credentials_path):
        """Authenticate with Google Search Console API"""
        credentials = Credentials.from_authorized_user_file(
            credentials_path,
            scopes=['https://www.googleapis.com/auth/webmasters.readonly']
        )
        return build('searchconsole', 'v1', credentials=credentials)
    
    def get_page_performance(self, page_url, start_date, end_date):
        """Get performance metrics for specific page"""
        request = {
            'startDate': start_date.strftime('%Y-%m-%d'),
            'endDate': end_date.strftime('%Y-%m-%d'),
            'dimensions': ['page', 'query'],
            'dimensionFilterGroups': [{
                'filters': [{
                    'dimension': 'page',
                    'operator': 'equals',
                    'expression': page_url
                }]
            }],
            'rowLimit': 25000
        }
        
        try:
            response = self.service.searchanalytics().query(
                siteUrl=self.property_url,
                body=request
            ).execute()
            
            rows = response.get('rows', [])
            if not rows:
                return pd.DataFrame()
            
            data = []
            for row in rows:
                data.append({
                    'page': row['keys'][0],
                    'query': row['keys'][1],
                    'clicks': row['clicks'],
                    'impressions': row['impressions'],
                    'ctr': row['ctr'],
                    'position': row['position']
                })
            
            return pd.DataFrame(data)
        
        except Exception as e:
            print(f"Error fetching data: {e}")
            return pd.DataFrame()
    
    def compare_periods(self, pages, baseline_weeks=4, test_weeks=8):
        """Compare baseline vs test period"""
        today = datetime.now()
        
        # Baseline period (before test)
        baseline_end = today - timedelta(weeks=test_weeks)
        baseline_start = baseline_end - timedelta(weeks=baseline_weeks)
        
        # Test period (during test)
        test_end = today
        test_start = today - timedelta(weeks=test_weeks)
        
        results = []
        
        for page in pages:
            print(f"Analyzing {page}...")
            
            baseline_data = self.get_page_performance(
                page, baseline_start, baseline_end
            )
            test_data = self.get_page_performance(
                page, test_start, test_end
            )
            
            if baseline_data.empty or test_data.empty:
                continue
            
            baseline_clicks = baseline_data['clicks'].sum()
            test_clicks = test_data['clicks'].sum()
            baseline_impressions = baseline_data['impressions'].sum()
            test_impressions = test_data['impressions'].sum()
            
            # Calculate changes
            click_change = ((test_clicks - baseline_clicks) / baseline_clicks * 100) if baseline_clicks > 0 else 0
            ctr_baseline = (baseline_clicks / baseline_impressions * 100) if baseline_impressions > 0 else 0
            ctr_test = (test_clicks / test_impressions * 100) if test_impressions > 0 else 0
            ctr_change = ctr_test - ctr_baseline
            
            position_baseline = baseline_data['position'].mean()
            position_test = test_data['position'].mean()
            position_change = position_test - position_baseline
            
            results.append({
                'page': page,
                'baseline_clicks': baseline_clicks,
                'test_clicks': test_clicks,
                'click_change_pct': round(click_change, 2),
                'baseline_ctr': round(ctr_baseline, 2),
                'test_ctr': round(ctr_test, 2),
                'ctr_change': round(ctr_change, 2),
                'baseline_position': round(position_baseline, 2),
                'test_position': round(position_test, 2),
                'position_change': round(position_change, 2)
            })
        
        return pd.DataFrame(results)
    
    def calculate_statistical_significance(self, control_group, variant_group):
        """Calculate if results are statistically significant"""
        from scipy import stats
        
        control_clicks = control_group['test_clicks'].sum()
        control_impressions = control_group['baseline_clicks'].sum()  # Proxy for impressions
        variant_clicks = variant_group['test_clicks'].sum()
        variant_impressions = variant_group['baseline_clicks'].sum()
        
        # Chi-square test
        contingency_table = np.array([
            [control_clicks, control_impressions - control_clicks],
            [variant_clicks, variant_impressions - variant_clicks]
        ])
        
        chi2, p_value, dof, expected = stats.chi2_contingency(contingency_table)
        
        return {
            'chi_square': chi2,
            'p_value': p_value,
            'significant': p_value < 0.05,
            'confidence_level': (1 - p_value) * 100 if p_value < 1 else 0
        }

# Usage Example

    # Initialize tester
    tester = GSCSEOTester(
        property_url='https://example.com',
        credentials_path='credentials.json'

        'https://example.com/product/123',

        # ... more control pages
    ]
    
    variant_pages = [

    ]

    print(f"\nStatistical Significance:")

    print(f"  Significant: {significance['significant']}")
```

### Core Web Vitals: The UX-SEO Bridge

Google's Core Web Vitals represent the convergence point where UX and SEO align perfectly.

#### Largest Contentful Paint (LCP)

**Target**: < 2.5 seconds

**What it measures**: Loading performance - time until the largest content element is visible

**Real-World Impact**:
- SEO: Confirmed ranking factor since June 2021
- UX: Users abandon slow sites (53% leave if load >3s)
- CRO: Every 1s delay = 7% reduction in conversions

#### First Input Delay (FID) / Interaction to Next Paint (INP)

**Targets**: 
- FID: < 100ms (being phased out)
- INP: < 200ms (replacement metric, March 2024)

**What they measure**: 
- FID: Time from first interaction to browser response
- INP: Worst interaction latency across entire page visit

**Optimization Strategy**:

```javascript
// Problem: Heavy synchronous JavaScript blocks main thread
// BAD APPROACH
function processLargeDataset(data) {
  const results = [];
  
  // This blocks the main thread for several seconds
  for (let i = 0; i < data.length; i++) {
    results.push(expensiveComputation(data[i]));
  }
  
  displayResults(results);
}

// User clicks button during processing
// → Browser is frozen
// → Poor FID/INP score
// → Terrible user experience

// GOOD APPROACH 1: Break into chunks with timeouts
async function processLargeDatasetChunked(data) {
  const chunkSize = 100;
  const results = [];
  
  for (let i = 0; i < data.length; i += chunkSize) {
    const chunk = data.slice(i, i + chunkSize);
    
    // Process chunk
    for (let item of chunk) {
      results.push(expensiveComputation(item));
    }
    
    // Yield to main thread (allow interactions)
    await new Promise(resolve => setTimeout(resolve, 0));
    
    // Update progress
    updateProgressBar((i / data.length) * 100);
  }
  
  displayResults(results);
}

// GOOD APPROACH 2: Use Web Workers (even better)
// main.js
const worker = new Worker('data-processor.js');
const progressBar = document.getElementById('progress');

worker.postMessage({ data: largeDataset });

worker.onmessage = function(e) {
  if (e.data.type === 'progress') {
    progressBar.value = e.data.percent;
  } else if (e.data.type === 'complete') {
    displayResults(e.data.results);
  }
};

// data-processor.js (Web Worker)
self.onmessage = function(e) {
  const data = e.data.data;
  const results = [];
  
  for (let i = 0; i < data.length; i++) {
    results.push(expensiveComputation(data[i]));
    
    // Report progress every 10%
    if (i % Math.floor(data.length / 10) === 0) {
      self.postMessage({
        type: 'progress',
        percent: (i / data.length) * 100
      });
    }
  }
  
  self.postMessage({
    type: 'complete',
    results: results
  });
};

// GOOD APPROACH 3: Use requestIdleCallback for non-urgent tasks
function processWhenIdle(tasks) {
  function processTasks(deadline) {
    while (deadline.timeRemaining() > 0 && tasks.length > 0) {
      const task = tasks.shift();
      processTask(task);
    }
    
    if (tasks.length > 0) {
      requestIdleCallback(processTasks);
    }
  }
  
  requestIdleCallback(processTasks);
}
```

```javascript
class OptimizedProductFilter {
  constructor(products) {
    this.products = products;
    this.filterWorker = new Worker('filter-worker.js');
    this.setupWorker();
  }
  
  setupWorker() {
    this.filterWorker.onmessage = (e) => {
      this.renderProducts(e.data.products);
    };
  }
  
  async filterProducts(maxPrice) {
    // Show loading state immediately (feels responsive)
    this.showLoadingState();
    
    // Offload heavy computation to worker
    this.filterWorker.postMessage({
      products: this.products,
      maxPrice: maxPrice
    });
    
    // Main thread remains responsive
    // User can still interact with page
  }
  
  renderProducts(products) {
    const container = document.getElementById('product-grid');
    
    // Render in chunks to avoid blocking
    this.renderInChunks(container, products, 20);
  }
  
  async renderInChunks(container, products, chunkSize) {
    container.innerHTML = '';
    
    for (let i = 0; i < products.length; i += chunkSize) {
      const chunk = products.slice(i, i + chunkSize);
      const fragment = document.createDocumentFragment();
      
      chunk.forEach(product => {
        const element = this.createProductElement(product);
        fragment.appendChild(element);
      });
      
      container.appendChild(fragment);
      
      // Yield to main thread
      await new Promise(resolve => setTimeout(resolve, 0));
    }
  }
  
  showLoadingState() {
    document.getElementById('product-grid').innerHTML = `
      <div class="loading-skeleton">
        ${Array(20).fill('<div class="skeleton-card"></div>').join('')}
      </div>
    `;
  }
  
  createProductElement(product) {
    const div = document.createElement('div');
    div.className = 'product-card';
    div.innerHTML = `
      <img src="${product.image}" loading="lazy" alt="${product.name}">
      <h3>${product.name}</h3>
      <p class="price">$${product.price}</p>
      <button data-product-id="${product.id}">Add to Cart</button>
    `;
    return div;
  }
}
```

#### Cumulative Layout Shift (CLS)

**Target**: < 0.1

**What it measures**: Visual stability - unexpected layout shifts during page load

**Common Causes**:
1. Images without dimensions
2. Ads, embeds, iframes without reserved space
3. Dynamically injected content
4. Web fonts causing FOIT/FOUT
5. Actions waiting for network response

**Optimization Strategy**:

```html
<!-- PROBLEM 1: Images without dimensions -->
<!-- BAD -->
<img src="product.jpg" alt="Product">
<!-- Image loads → Page jumps down → Poor CLS -->

<!-- GOOD: Set explicit dimensions -->
<img src="product.jpg" alt="Product" width="800" height="600">

<!-- BETTER: Use aspect-ratio for responsive images -->
<div class="image-container" style="aspect-ratio: 4/3;">
  <img src="product.jpg" 
       alt="Product" 
       style="width: 100%; height: 100%; object-fit: cover;">
</div>

<!-- BEST: Modern responsive with aspect-ratio -->
<picture>
  <source media="(min-width: 1200px)" srcset="product-lg.webp" width="1200" height="900">
  <source media="(min-width: 768px)" srcset="product-md.webp" width="800" height="600">
  <img src="product-sm.webp" 
       alt="Product" 
       width="400" 
       height="300"
       style="aspect-ratio: 4/3; width: 100%; height: auto;">
</picture>
```


```css
/* BAD: Default font loading */

  font-family: 'CustomFont';
  src: url('font.woff2') format('woff2');

}

  font-display: optional; /* Only use if already cached */
```

```javascript
// Google-Safe Mobile Popup Implementation
class MobilePopupManager {
  constructor() {
    this.hasInteracted = false;
    this.scrollDepth = 0;
    this.timeOnPage = 0;
    this.isMobile = window.innerWidth < 768;
  }
  
  init() {
    if (!this.isMobile) {
      // Desktop: More flexibility
      this.initDesktopPopup();
    } else {
      // Mobile: Strict Google guidelines
      this.initMobileCompliantPopup();
    }
  }
  
  initMobileCompliantPopup() {
    // Track user engagement
    document.addEventListener('touchstart', () => {
      this.hasInteracted = true;
    }, { once: true });
    
    window.addEventListener('scroll', () => {
      this.scrollDepth = (window.scrollY / document.body.scrollHeight) * 100;
    });
    
    // Check every 10 seconds
    setInterval(() => {
      this.timeOnPage += 10;
      this.evaluatePopupDisplay();
    }, 10000);
  }
  
  evaluatePopupDisplay() {
    // Google-compliant conditions:
    // 1. User must interact first
    // 2. Significant scroll depth (60%+)
    // 3. Minimum time on page (30s+)
    // 4. Popup must not cover main content
    
    if (this.hasInteracted && 
        this.scrollDepth > 60 && 
        this.timeOnPage >= 30 &&
        !this.hasShownPopup) {
      
      this.showCompliantPopup();
      this.hasShownPopup = true;
    }
  }
  
  showCompliantPopup() {
    // Small slide-in (not full-screen overlay)
    const popup = document.createElement('div');
    popup.className = 'compliant-popup';
    popup.innerHTML = `
      <div class="popup-content">
        <button class="close-btn" aria-label="Close">&times;</button>
        <h3>Join Our Newsletter</h3>
        <p>Get weekly tips delivered to your inbox</p>
        <form>
          <input type="email" placeholder="your@email.com" required>
          <button type="submit">Subscribe</button>
        </form>
      </div>
    `;
    
    document.body.appendChild(popup);
    
    // Close functionality
    popup.querySelector('.close-btn').addEventListener('click', () => {
      popup.remove();
    });
  }
  
  initDesktopPopup() {
    // Desktop: More aggressive options available
    setTimeout(() => {
      if (this.scrollDepth > 40) {
```

```python
# SEO-CRO Test Prioritization Calculator
class TestPrioritization:
    def __init__(self):
        self.weights = {
            'impact': 0.4,      # Potential business impact
            'effort': 0.3,      # Implementation difficulty (inverse)
            'risk': 0.2,        # Risk level (inverse)
            'speed': 0.1        # Time to results (inverse)
        }
    
    def calculate_score(self, test):
        """
        Calculate priority score for a test
        
        Scores (1-10 scale):
        - potential_impact: Expected improvement (10 = transformative)
        - implementation_effort: How hard to implement (10 = very difficult)
        - risk_level: Chance of negative impact (10 = very risky)
        - time_to_results: Weeks until measurable results (10 = >12 weeks)
        """
        impact_score = test['potential_impact']
        effort_score = 10 - test['implementation_effort']  # Inverse
        risk_score = 10 - test['risk_level']  # Inverse
        speed_score = 10 - test['time_to_results']  # Inverse
        
        total_score = (
            impact_score * self.weights['impact'] +
            effort_score * self.weights['effort'] +
            risk_score * self.weights['risk'] +
            speed_score * self.weights['speed']
        )
        
        return round(total_score, 2)
    
    def prioritize_tests(self, tests):
        """Sort tests by priority score"""
        for test in tests:
            test['priority_score'] = self.calculate_score(test)
        
        return sorted(tests, key=lambda x: x['priority_score'], reverse=True)
    
    def generate_report(self, tests):
        """Generate prioritization report"""
        sorted_tests = self.prioritize_tests(tests)
        
        print("=" * 80)
        print("TEST PRIORITIZATION REPORT")
        print("=" * 80)

            print(f"{i}. {test['name']} (Score: {test['priority_score']}/10)")
            print(f"   Impact: {test['potential_impact']}/10")

            print(f"   Time to Results: {test['time_to_results']} weeks")
```

```sql
WITH gsc_data AS (

    page,

    AVG(position) as avg_position,

  WHERE date BETWEEN '2024-01-01' AND '2024-12-31'
  GROUP BY page
),

  SELECT
    landing_page,
    COUNT(DISTINCT user_pseudo_id) as sessions,
    SUM(CASE WHEN event_name = 'purchase' THEN 1 ELSE 0 END) as conversions,
    AVG(engagement_time_msec) / 1000 as avg_time_on_page,
    (SUM(CASE WHEN event_name = 'purchase' THEN 1 ELSE 0 END) / 
     COUNT(DISTINCT user_pseudo_id)) * 100 as conversion_rate,
    SUM(ecommerce.purchase_revenue_in_usd) as revenue
  FROM `project.dataset.ga4_events`
  WHERE event_date BETWEEN '20240101' AND '20241231'
    AND traffic_source.medium = 'organic'
  GROUP BY landing_page
),

cwv_data AS (
  SELECT
    page_url,
    AVG(CAST(lcp_value AS FLOAT64)) as avg_lcp,
    AVG(CAST(fid_value AS FLOAT64)) as avg_fid,
    AVG(CAST(cls_value AS FLOAT64)) as avg_cls,
    SUM(CASE WHEN lcp_value < 2500 THEN 1 ELSE 0 END) / COUNT(*) as lcp_good_pct,
    SUM(CASE WHEN fid_value < 100 THEN 1 ELSE 0 END) / COUNT(*) as fid_good_pct,
    SUM(CASE WHEN cls_value < 0.1 THEN 1 ELSE 0 END) / COUNT(*) as cls_good_pct
  FROM `project.dataset.crux_data`
  WHERE date BETWEEN '2024-01-01' AND '2024-12-31'
  GROUP BY page_url
)

SELECT
  COALESCE(gsc.page, ga4.landing_page, cwv.page_url) as page,
  
  -- SEO Metrics
  gsc.organic_clicks,
  gsc.impressions,
  gsc.avg_position,
  gsc.organic_ctr,
  
  -- CRO Metrics
  ga4.sessions,
  ga4.conversions,

  cwv.avg_cls,
  cwv.lcp_good_pct,

  cwv.cls_good_pct,
  
  -- Opportunity Scoring
```

### Case Study 1: SaaS Company - Balancing Free Trial Optimization

**Company**: Project Management Software
**Vertical**: B2B SaaS
**Challenge**: Increase free trial signups without hurting SEO rankings

**Initial State**:
- 50,000 monthly organic visitors
- 2.3% free trial conversion rate (1,150 signups/month)
- Average ranking: Position 8 for 50 target keywords
- 200 blog posts driving most traffic
- High bounce rate on blog content (72%)
- Avg. time on page: 1:45

**Conflict Identified**:
CRO team wanted aggressive CTAs throughout blog content to drive trial signups. SEO team worried this would:
- Increase bounce rate (negative UX signal)
- Reduce time on page (engagement metric)
- Damage content quality perception
- Hurt rankings for informational queries

**Research Phase (Weeks 1-2)**:

1. **Heatmap Analysis** (Hotjar):
   - Only 23% of visitors scrolled past 40% of article
   - Most exits happened at first CTA (appearing at 15% scroll)
   - Very few clicks on sidebar CTAs

2. **Exit Surveys** (Qualaroo):
   - 68% of respondents said content was "too sales-y"
   - 41% came for information, not ready to buy
   - Top request: "More helpful content, less selling"

3. **Competitor Analysis**:
   - Top-ranking competitors had less aggressive CTAs
   - CTAs appeared naturally after 60-70% content consumption
   - Softer language: "Learn more" vs "Start free trial now"

4. **User Session Recording Analysis**:
   - Users who converted spent avg. 8:30 on content before clicking CTA
   - Early CTAs were actively scrolled past or closed
   - Engaged users naturally looked for next steps

**Hypothesis (Week 3)**:
*"If we reduce CTA aggression in the first 60% of content and implement engagement-based smart CTAs, we'll increase both conversion rates AND improve SEO metrics (time on page, bounce rate), leading to better rankings."*

**Implementation (Week 4)**:

```javascript
// Smart CTA System Implementation
class EngagementBasedCTASystem {
  constructor() {
    this.engagementScore = 0;
    this.triggers = {
      scrollDepth: false,      // Must scroll 60%+
      timeOnPage: false,       // Must spend 45s+
      interactions: false,     // Must interact (click, highlight, etc.)
      readingPattern: false    // Must show reading behavior
    };
    this.maxCTAsShown = 0;
    this.ctaShown = false;
  }
  
  init() {
    this.trackEngagement();
    this.evaluatePerio dically();
  }
  
  trackEngagement() {
    // Scroll depth tracking
    let lastScrollY = 0;
    window.addEventListener('scroll', () => {
      const scrollPercent = (window.scrollY / document.body.scrollHeight) * 100;
      
      if (scrollPercent > 60) {
        this.triggers.scrollDepth = true;
      }
      
      // Detect reading pattern (slow, consistent scrolling)
      const scrollSpeed = Math.abs(window.scrollY - lastScrollY);
      if (scrollSpeed < 100) {  // Slow scroll = reading
        this.triggers.readingPattern = true;
      }
      lastScrollY = window.scrollY;
    });
    
    // Time on page
    setTimeout(() => {
      this.triggers.timeOnPage = true;
    }, 45000);  // 45 seconds
    
    // Interactions
    ['click', 'mouseup', 'keyup'].forEach(event => {
      document.addEventListener(event, () => {
        this.triggers.interactions = true;
      }, { once: true });
    });
  }
  
  evaluatePerio dically() {
    setInterval(() => {
      this.checkEngagement();
    }, 5000);  // Check every 5 seconds
  }
  
  checkEngagement() {
    const triggeredCount = Object.values(this.triggers)
      .filter(v => v === true).length;
    
    // Calculate engagement score (0-100)
    this.engagementScore = (triggeredCount / Object.keys(this.triggers).length) * 100;
    
    // Show CTA if high engagement and not already shown
    if (this.engagementScore >= 75 && !this.ctaShown) {
      this.showSmartCTA();
    }
  }
  
  showSmartCTA() {
    this.ctaShown = true;
    
    const cta = document.createElement('div');
    cta.className = 'smart-cta slide-in';
    cta.innerHTML = `
      <div class="cta-content">
        <button class="close-cta" aria-label="Close">×</button>
        <h3>Ready to get started?</h3>
        <p>Try our project management tool free for 14 days.</p>
        <a href="/signup" class="cta-button">Start Free Trial</a>
        <p class="cta-subtext">No credit card required</p>
      </div>
    `;
    
    document.body.appendChild(cta);
    
    // Track CTA interaction
    this.trackCTAPerformance(cta);
    
    // Close button
    cta.querySelector('.close-cta').addEventListener('click', () => {
      cta.remove();
    });
  }
  
  trackCTAPerformance(cta) {
    // Track impressions
    if (window.gtag) {
      gtag('event', 'cta_impression', {
        'cta_type': 'smart_cta',
        'engagement_score': this.engagementScore
      });
    }
    
    // Track clicks
    cta.querySelector('.cta-button').addEventListener('click', () => {
      if (window.gtag) {
        gtag('event', 'cta_click', {
          'cta_type': 'smart_cta',
          'engagement_score': this.engagementScore
        });
      }
    });
  }
}

// Initialize on blog pages
if (document.body.classList.contains('blog-post')) {
  new EngagementBasedCTASystem().init();
}
```

### Case Study 3: News Publisher - Core Web Vitals vs Ad Revenue

**Company**: Digital News Publisher
**Challenge**: Improve Core Web Vitals without sacrificing ad revenue

**Initial State**:
- CLS: 0.48 (Poor)
- LCP: 3.8s (Poor)
- Ad revenue: $50K/month
- Organic traffic declining 15% YoY

**Solution**:
- Reserved space for all ad units
- Lazy-loaded below-fold ads
- Optimized ad refresh rates
- Implemented ad placeholder design

**Results**:
- CLS: 0.48 → 0.05 (Pass)
- LCP: 3.8s → 1.9s (Pass)
- Ad revenue: Maintained at $49K/month (-2%)
- Organic traffic: +34% over 3 months
- Net revenue: +$18.5K/month (+37%)

**Key Insight**: Small ad revenue sacrifice (2%) was more than offset by traffic gains (34%), resulting in 37% net revenue increase.

## Conclusion

Balancing SEO and CRO is not about choosing one over the other—it's about finding strategic alignment. The most successful digital marketers:

1. **Prioritize Core Web Vitals** - The clearest intersection of SEO and CRO
2. **Test methodically** - Use proper SEO testing frameworks
3. **Measure holistically** - Track both traffic AND conversion metrics
4. **Think long-term** - Short-term conversion gains aren't worth long-term visibility loss
5. **Find win-wins** - 80% of value comes from initiatives that help both

Remember: **More traffic means nothing if it doesn't convert. High conversion rates mean nothing if nobody finds your site.**

The goal is **sustainable, profitable growth** through the strategic balance of visibility and conversion optimization.
