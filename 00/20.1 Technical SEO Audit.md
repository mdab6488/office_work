## Introduction to Technical SEO Audit

A Technical SEO Audit examines the technical infrastructure of a website to identify issues that prevent search engines from crawling, indexing, and ranking your content effectively.

### Why Technical SEO Audits Matter

**Real-World Impact:**
- **Case Study 1:** E-commerce site TechGear.com fixed crawl errors and improved site speed, resulting in 45% increase in organic traffic within 3 months
- **Case Study 2:** SaaS company CloudTools identified 3,000+ duplicate pages causing indexation bloat, removed them, and saw 67% improvement in ranking positions
- **Case Study 3:** News website DailyNews.net implemented HTTPS and fixed mobile issues, leading to 89% increase in mobile organic traffic

### Audit Frequency
- **Small Sites (< 1,000 pages):** Quarterly audits
- **Medium Sites (1,000-10,000 pages):** Monthly audits
- **Large Sites (> 10,000 pages):** Weekly monitoring + Monthly comprehensive audits
- **E-commerce Sites:** Bi-weekly audits during peak seasons

## Crawlability Check

### What is Crawlability?

Crawlability refers to search engine bots' ability to access and navigate your website's pages. If bots can't crawl your site, your pages won't be indexed or ranked.

### Essential Tools for Crawlability Analysis

1. **Screaming Frog SEO Spider** (Desktop crawler)
2. **Google Search Console** (Coverage report)
3. **Ahrefs Site Audit** (Cloud-based crawler)
4. **Sitebulb** (Visual crawler)
5. **DeepCrawl/Lumar** (Enterprise solution)

### 1. Robots.txt Analysis

**What to Check:**
```
# Access your robots.txt
https://yourdomain.com/robots.txt
```

**Common Issues & Real Examples:**

**Example 1: Accidentally Blocking Important Pages**
```
# BAD - Blocks entire site
User-agent: *
Disallow: /

# GOOD - Strategic blocking
User-agent: *
Disallow: /admin/
Disallow: /cart/
Disallow: /checkout/
Disallow: /*?sort=*
Disallow: /*?filter=*
Allow: /
Sitemap: https://yourdomain.com/sitemap.xml
```

**Real-Life Case:**
Fashion retailer StyleHub.com accidentally blocked their entire `/products/` directory in robots.txt during a site update. They lost 92% of organic traffic within 2 weeks. After fixing robots.txt, recovery took 6 weeks.

**How to Test Robots.txt:**
1. Go to Google Search Console > Settings > robots.txt Tester
2. Enter URLs to test if they're blocked
3. Use Screaming Frog to identify robots.txt blocked URLs

**Best Practices:**
- Always use `Allow:` directive to override blanket `Disallow:` rules
- Block duplicate content (sorting, filtering, search results)
- Block admin, user account, and shopping cart pages
- Never block CSS/JS files (Google needs them to render pages)
- Include sitemap location

### 2. XML Sitemap Validation

**What to Check:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://www.example.com/page-1</loc>
    <lastmod>2025-11-10</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.8</priority>
  </url>
</urlset>
```

**Common Sitemap Errors:**

**Example 1: Including Blocked URLs**
```
❌ BAD: Sitemap contains 500 URLs that are:
- Blocked by robots.txt
- 404 errors
- 301 redirects
- Noindexed pages
- Canonicalized to other URLs

✅ GOOD: Sitemap contains only:
- Indexable pages (HTTP 200 status)
- Self-canonical pages
- No robots.txt blocks
- No noindex tags
```

**Real-Life Example:**
B2B software company DataPro had a sitemap with 8,500 URLs, but only 2,100 were valid. The remaining URLs included:
- 3,200 URLs blocked by robots.txt
- 1,800 404 error pages
- 1,100 redirect chains
- 300 duplicate content pages

**Action taken:** Created clean sitemap with only valid URLs, resubmitted to GSC, saw 34% increase in indexed pages within 4 weeks.

**Sitemap Audit Process:**
```bash
# 1. Download sitemap
wget https://yourdomain.com/sitemap.xml

# 2. Validate XML structure
xmllint --noout sitemap.xml

# 3. Extract URLs and check status codes
curl -I https://yourdomain.com/page-1

# 4. Use Screaming Frog to crawl sitemap URLs
# File > Crawl > Upload Sitemap
```

**Sitemap Best Practices:**
- Split large sitemaps (max 50,000 URLs per file)
- Create sitemap index file for multiple sitemaps
- Update lastmod dates accurately
- Include only canonical, indexable URLs
- Submit to Google Search Console and Bing Webmaster Tools

### 3. Internal Link Structure Analysis

**Why It Matters:**
Internal links distribute PageRank, help crawlers discover pages, and establish site hierarchy.

**Key Metrics to Check:**

1. **Orphan Pages** (pages with no internal links)
2. **Link Depth** (clicks from homepage)
3. **Broken Internal Links** (404s)
4. **Redirect Chains** (multiple 301s)
5. **Nofollow Internal Links** (wasted PageRank)

**Real-Life Example: E-commerce Site**

**Before Audit:**
- 2,847 orphan product pages (no internal links)
- Average link depth: 8 clicks from homepage
- 634 broken internal links
- 312 redirect chains (averaging 4 redirects each)

**Issues Identified:**
```
Homepage → Category → Subcategory → Products Page → Product
(1 click)   (2 clicks)  (3 clicks)      (4 clicks)     (5 clicks)

But some products required:
Homepage → ... → 12 clicks → Product
```

**Solutions Implemented:**
```html
<!-- Added breadcrumbs -->
<nav aria-label="Breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/mens-clothing">Men's Clothing</a></li>
    <li><a href="/mens-clothing/shirts">Shirts</a></li>
    <li>Blue Oxford Shirt</li>
  </ol>
</nav>

<!-- Added related products section -->
<section class="related-products">
  <h2>You May Also Like</h2>
  <a href="/mens-clothing/shirts/white-oxford">White Oxford Shirt</a>
  <a href="/mens-clothing/shirts/pink-dress">Pink Dress Shirt</a>
  <a href="/mens-clothing/shirts/striped-casual">Striped Casual Shirt</a>
</section>

<!-- Added faceted navigation -->
<aside class="filters">
  <a href="/mens-clothing/shirts?size=small">Small</a>
  <a href="/mens-clothing/shirts?size=medium">Medium</a>
  <a href="/mens-clothing/shirts?color=blue">Blue</a>
</aside>
```

**Results After 3 Months:**
- Orphan pages reduced to 47 (98% reduction)
- Average link depth: 3.2 clicks
- All broken links fixed
- Redirect chains eliminated
- **41% increase in product page rankings**
- **28% increase in organic revenue**

**How to Find Internal Link Issues:**

Using Screaming Frog:
```
1. Crawl your site
2. Internal > All
3. Filter by:
   - Status Code = 404 (broken links)
   - Status Code = 301/302 (redirects)
   - Inlinks = 0 (orphan pages)
4. Bulk Export > Internal Links > Export
```

Using Google Search Console:
```
Links > Internal links > Sort by "Target page"
Identify pages with unusually low internal links
```

### 4. Crawl Budget Optimization

**What is Crawl Budget?**
The number of pages Googlebot will crawl on your site in a given time period.

**Factors That Waste Crawl Budget:**

**Real-Life Example: Large News Website**

**Problem:** DailyNews.net had 500,000 pages but Google only crawled 50,000/day
- 200,000 duplicate content pages
- 150,000 low-quality tag/archive pages
- 80,000 URL parameter variations
- 70,000 outdated/irrelevant pages

**Crawl Budget Audit Results:**
```
Total Pages on Site: 500,000
Pages Crawled (30 days): 1,200,000 crawls
Unique Pages Crawled: 280,000
Important Pages: 50,000
Important Pages Crawled: 35,000 (70% coverage)

Wasted Crawl Budget:
- Duplicate pages: 450,000 crawls (37.5%)
- URL parameters: 380,000 crawls (31.7%)
- Low-quality archives: 285,000 crawls (23.8%)
- 404 errors: 85,000 crawls (7%)
```

**Solutions Implemented:**

1. **Robots.txt Optimization**
```
User-agent: *
Disallow: /*?sort=*
Disallow: /*?filter=*
Disallow: /*?page=*
Disallow: /tag/
Disallow: /archive/
Disallow: /search/
Allow: /
```

2. **URL Parameter Handling** (Google Search Console)
```
Go to: Legacy tools and reports > URL Parameters
Set crawl behavior:
- sort = Let Googlebot decide
- filter = No URLs
- page = Representative URL
```

3. **Canonical Tags for Pagination**
```html
<!-- Page 2 of article archive -->
<link rel="canonical" href="https://dailynews.net/category/politics/">

<!-- Or use self-referencing canonical for pagination -->
<link rel="canonical" href="https://dailynews.net/category/politics/?page=2">
<link rel="prev" href="https://dailynews.net/category/politics/">
<link rel="next" href="https://dailynews.net/category/politics/?page=3">
```

4. **Noindex Low-Value Pages**
```html
<!-- Tag pages, search results, filters -->
<meta name="robots" content="noindex, follow">
```

**Results After 2 Months:**
- Crawl budget focused on important pages: 92% coverage
- Important page crawl frequency: 2.3x increase
- New articles indexed within 4 hours (vs 48 hours before)
- **37% increase in organic traffic to new content**
- **52% increase in Google Discover traffic**

### 5. JavaScript Rendering Check

**Why It Matters:**
Google renders JavaScript, but it's a two-step process that can delay indexing.

**How Google Crawls JavaScript:**
```
Step 1: Initial Crawl (HTML only)
↓
Step 2: Rendering Queue (Days or weeks later)
↓
Step 3: JavaScript Execution & Content Discovery
```

**Real-Life Example: React E-commerce Site**

**Problem:** TechStore.com built with React (client-side rendering)
- Product listings loaded via JavaScript
- Product names/descriptions not in initial HTML
- Critical SEO elements missing from source code

**Testing JavaScript Rendering:**

1. **View Page Source** (Ctrl+U)
```html
<!-- What Google sees initially -->
<body>
  <div id="root"></div>
  <!-- No product content visible! -->
</body>
```

2. **Inspect Element** (Ctrl+Shift+I)
```html
<!-- What browser renders after JavaScript executes -->
<body>
  <div id="root">
    <h1>Blue Widget Pro</h1>
    <p>Advanced widget with premium features...</p>
    <div class="price">$99.99</div>
  </div>
</body>
```

**Problem Identified:** Critical SEO content not in initial HTML!

**Testing Tools:**

1. **Google Search Console URL Inspection**
```
1. Enter URL
2. Test Live URL
3. View Crawled Page
4. Compare HTML vs Rendered HTML
```

2. **Mobile-Friendly Test Tool**
```
https://search.google.com/test/mobile-friendly
Shows rendered version + screenshot
```

3. **JavaScript SEO Testing**
```bash
# Fetch as Googlebot
curl -A "Googlebot" https://yourdomain.com/page

# Compare with browser fetch
curl -A "Mozilla/5.0" https://yourdomain.com/page
```

**Solutions Implemented:**

**Option 1: Server-Side Rendering (SSR)**
```javascript
// Next.js implementation
export async function getServerSideProps() {
  const products = await fetchProducts();
  return { props: { products } };
}

export default function ProductPage({ products }) {
  return (
    <div>
      <h1>{products.name}</h1>
      <p>{products.description}</p>
    </div>
  );
}
```

**Option 2: Dynamic Rendering (Prerendering)**
```javascript
// Detect Googlebot and serve pre-rendered HTML
const userAgent = req.headers['user-agent'];
if (/googlebot|bingbot/i.test(userAgent)) {
  return prerenderService.render(req.url);
}
```

**Option 3: Static Site Generation (SSG)**
```javascript
// Next.js - Build HTML at build time
export async function getStaticProps() {
  const products = await fetchProducts();
  return { props: { products } };
}
```

**Results:**
- 100% of SEO content in initial HTML
- Indexing speed: 6 hours (vs 3-4 weeks before)
- Product page rankings improved by 2.3 positions on average
- **89% increase in organic product traffic**

**JavaScript SEO Best Practices:**
- Include critical content in initial HTML
- Use semantic HTML elements (h1, h2, p, etc.)
- Implement lazy loading for below-fold content
- Test with JavaScript disabled
- Monitor JavaScript errors in Google Search Console

### 6. Server Response Time & Status Codes

**HTTP Status Codes & SEO Impact:**

**Real-World Status Code Issues:**

**Example 1: Soft 404 Errors**
```
URL: https://shop.com/product/deleted-item
Status Code: 200 OK
Content: "Product not found"

Problem: Returns 200 instead of 404
Impact: Wastes crawl budget, confuses search engines
```

**Correct Implementation:**
```php
<?php
if (!productExists($productId)) {
    header("HTTP/1.1 404 Not Found");
    include('404.php');
    exit;
}
?>
```

**Example 2: Redirect Chains**
```
Real-Life Case: FashionOutlet.com

URL 1: http://fashionoutlet.com/mens-shoes
→ 301 to https://fashionoutlet.com/mens-shoes
→ 301 to https://www.fashionoutlet.com/mens-shoes
→ 301 to https://www.fashionoutlet.com/mens-shoes/
→ 301 to https://www.fashionoutlet.com/men/footwear/
→ 200 OK

Result: 5 redirects = Lost PageRank, slow load time
```

**How to Find Redirect Chains:**

Using Screaming Frog:
```
1. Crawl site
2. Response Codes > Redirection (3xx)
3. Sort by "Redirect Chain"
4. Filter: Chain Length > 1
```

**Solution:**
```apache
# .htaccess - Direct redirect
RedirectMatch 301 ^/mens-shoes/?$ https://www.fashionoutlet.com/men/footwear/
```

**Results:**
- Eliminated 2,847 redirect chains
- Average page load time: 2.1s → 0.8s
- Rankings improved for 67% of affected pages
- **23% increase in organic traffic from redirected pages**

### 7. Pagination & Infinite Scroll

**Real-Life Example: E-commerce Category Pages**

**Problem:** ElectronicsHub.com used infinite scroll without proper implementation
- Only page 1 indexed
- Deep products never discovered by Google
- Category pages showed minimal products in HTML

**Infinite Scroll Issues:**
```html
<!-- Only 20 products in initial HTML -->
<div class="product-grid">
  <!-- Products 1-20 -->
</div>

<!-- JavaScript loads more on scroll -->
<script>
  window.addEventListener('scroll', loadMoreProducts);
</script>
```

**Google's View:**
- Sees only first 20 products
- Doesn't trigger scroll events
- Misses 80% of product catalog

**Solution 1: Paginated URLs**
```html
<!-- Implement traditional pagination -->
<nav class="pagination">
  <a href="/category/electronics?page=1">1</a>
  <a href="/category/electronics?page=2">2</a>
  <a href="/category/electronics?page=3">3</a>
</nav>

<!-- With rel="prev/next" tags -->
<link rel="prev" href="/category/electronics?page=1">
<link rel="canonical" href="/category/electronics?page=2">
<link rel="next" href="/category/electronics?page=3">
```

**Solution 2: "Load More" Button + View All**
```html
<!-- Infinite scroll for users -->
<button id="load-more">Load More Products</button>

<!-- SEO fallback -->
<noscript>
  <a href="/category/electronics?page=2">Next Page</a>
</noscript>

<!-- Provide "View All" option -->
<a href="/category/electronics/all">View All 500 Products</a>
```

**Solution 3: Hybrid Approach (Recommended)**
```html
<!-- User Experience: Infinite scroll -->
<div class="product-grid" data-page="1">
  <!-- Products load automatically -->
</div>

<!-- SEO: Update URL on scroll -->
<script>
window.addEventListener('scroll', function() {
  if (shouldLoadMore()) {
    const nextPage = currentPage + 1;
    loadProducts(nextPage);
    // Update URL without reload
    history.pushState({}, '', '?page=' + nextPage);
  }
});
</script>

<!-- Canonical pagination -->
<link rel="prev" href="?page=1">
<link rel="canonical" href="?page=2">
<link rel="next" href="?page=3">
```

**Results:**
- All 500 products indexed (vs 20 before)
- Category page depth increased from 1 to 25 pages
- Product discovery: 2,400% increase
- **67% increase in organic product page traffic**

## Indexation Issues

### Understanding Indexation

Indexation is the process of adding web pages into Google's search index. Even if pages are crawlable, they might not be indexed due to various issues.

### Google Search Console Coverage Report Analysis

**Real-Life Example: SaaS Documentation Site**

**Initial Coverage Report:**
```
Valid (Indexed): 4,234 pages
Excluded: 18,967 pages
Errors: 1,203 pages
Total Submitted: 24,404 pages

Indexation Rate: 17.4% ❌
```

**Breaking Down Exclusions:**

1. **Duplicate, Google chose different canonical (8,234 pages)**
```
Issue: Self-referencing canonical conflicts with Google's choice

Example:
Your canonical: https://docs.site.com/guide/setup
Google's choice: https://docs.site.com/en/guide/setup

Cause: Language parameters creating duplicates
```

**Solution:**
```html
<!-- Consolidate language versions -->
<link rel="canonical" href="https://docs.site.com/guide/setup">
<link rel="alternate" hreflang="en" href="https://docs.site.com/en/guide/setup">
<link rel="alternate" hreflang="es" href="https://docs.site.com/es/guide/setup">
<link rel="alternate" hreflang="fr" href="https://docs.site.com/fr/guide/setup">
```

2. **Crawled - currently not indexed (4,567 pages)**
```
Issue: Low-quality content, thin pages, or crawl budget exhaustion

Example Pages:
- /tag/misc (87 words, no unique value)
- /category/uncategorized (duplicate of homepage)
- /author/john-doe (automatically generated)
```

**Solution:**
```html
<!-- Noindex low-value pages -->
<meta name="robots" content="noindex, follow">
```

3. **Discovered - currently not indexed (3,421 pages)**
```
Issue: Pages found in sitemap/links but not crawled yet

Common Causes:
- New pages (waiting in queue)
- Low-priority pages
- Server errors during crawl
```

**Solution:**
```
1. Request indexing via URL Inspection Tool (high-priority pages)
2. Improve internal linking to new pages
3. Increase crawl budget allocation
4. Fix any server errors
```

4. **Excluded by 'noindex' tag (1,289 pages)**
```
Issue: Intentional noindex or accidental noindex

Check:
- Are these pages supposed to be noindexed?
- Any accidental noindex on important pages?
```

**Real Case:** Developer accidentally left staging noindex tag on production site:
```html
<!-- Left in production by mistake -->
<meta name="robots" content="noindex, nofollow">
```

Impact: Entire site de-indexed in 3 days, 100% traffic loss

5. **Soft 404 (234 pages)**
```
Issue: Returns 200 status but contains "not found" content

Example:
Status: 200 OK
Content: "This product has been discontinued"
Title: "Product Not Found - Site.com"
```

**Fix:**
```php
<?php
if (!$product->exists()) {
    header("HTTP/1.1 404 Not Found");
    include('404-template.php');
    exit;
}
?>
```

6. **Page with redirect (445 pages)**
```
Issue: Sitemap or internal links pointing to redirect URLs

Example:
Sitemap contains: https://site.com/old-page
But it redirects to: https://site.com/new-page
```

**Solution:**
Update sitemap and internal links to point directly to final URL.

**After Fixes (3 Months Later):**
```
Valid (Indexed): 16,789 pages
Excluded: 4,234 pages
Errors: 67 pages
Total Submitted: 21,090 pages

Indexation Rate: 79.6% ✅
```

**Results:**
- **295% increase in indexed pages**
- **142% increase in organic traffic**
- **89% reduction in crawl errors**

### Index Bloat Detection & Cleanup

**What is Index Bloat?**
Having too many low-quality or duplicate pages indexed, diluting your site's overall quality.

**Real-Life Example: E-commerce Site with Faceted Navigation**

**Problem:** ShoeStore.com had 487,000 pages indexed but only 12,000 products

**Index Breakdown:**
```
Products: 12,000
Faceted navigation combinations: 475,000

Examples of duplicate pages:
- /shoes
- /shoes?size=10
- /shoes?color=red
- /shoes?size=10&color=red
- /shoes?color=red&size=10 (parameter order variation)
- /shoes?size=10&color=red&brand=nike
- /shoes?page=2&size=10&color=red
... (470,000+ more combinations)
```

**Detection Method:**
```
1. Google Search Console: Check indexed pages
   site:shoestore.com

2. Compare with actual page count:
   - Products: 12,000
   - Categories: 234
   - Info pages: 45
   - Total valuable pages: ~12,300

3. Index Bloat: 487,000 - 12,300 = 474,700 unnecessary pages
```

**Identifying Low-Value Indexed Pages:**

Using Google Search Console:
```
1. Performance > Pages
2. Sort by impressions (ascending)
3. Identify pages with:
   - < 10 impressions/month
   - High impressions but 0 clicks
   - Duplicate content
```

Using Screaming Frog:
```
1. Crawl site
2. Extract inlinks data
3. Filter pages with:
   - 0-1 internal links
   - Thin content (< 200 words)
   - Duplicate titles/meta descriptions
```

**Index Bloat Solutions:**

**1. Robots.txt for Faceted Navigation**
```
User-agent: *
Disallow: /*?*size=*
Disallow: /*?*color=*
Disallow: /*?*brand=*
Disallow: /*?*price=*
Disallow: /*?page=*
Allow: /
```

**2. Canonical Tags**
```html
<!-- All filtered pages canonical back to main category -->
<!-- URL: /shoes?size=10&color=red -->
<link rel="canonical" href="https://shoestore.com/shoes">
```

**3. Noindex for Parameter Pages**
```php
<?php
// Check if URL has filter parameters
if (isset($_GET['size']) || isset($_GET['color']) || isset($_GET['brand'])) {
    echo '<meta name="robots" content="noindex, follow">';
}
?>
```

**4. Deindex Request (Google Search Console)**
```
1. Identify outdated/duplicate URLs
2. Use "Removals" tool for temporary deindexing
3. Implement permanent noindex or 404
4. Submit updated sitemap
```

**Results After Cleanup:**
```
Before:
- Indexed pages: 487,000
- Products ranking: 23% of products on page 1
- Average position: 47
- Organic traffic: 45,000/month

After (4 months):
- Indexed pages: 13,200 (97% reduction)
- Products ranking: 67% of products on page 1
- Average position: 18
- Organic traffic: 127,000/month (182% increase)
```

**Business Impact:**
- Revenue increase: 214%
- Conversion rate: 2.1% → 3.4%
- Average order value: $78 → $94

### Canonicalization Issues

**Real-Life Canonical Tag Mistakes:**

**Example 1: Wrong Canonical Direction**
```html
<!-- Page A: Main product page -->
<link rel="canonical" href="https://shop.com/products/widget-pro">

<!-- Page B: Variant page -->
<link rel="canonical" href="https://shop.com/products/widget-pro?color=blue">
❌ WRONG: Canonical points to itself (variant should canonical to main)

<!-- CORRECT: Variant should point to main page -->
<link rel="canonical" href="https://shop.com/products/widget-pro">
```

**Example 2: Cross-Domain Canonical Confusion**
```html
<!-- YourSite.com accidentally canonicalizing to competitor -->
<link rel="canonical" href="https://competitorsite.com/similar-product">
❌ WRONG: Telling Google to index competitor instead of you!
```

**Real Case:** Development team copy-pasted product pages from competitor for testing, forgot to update canonicals. Lost 78% of product page traffic before discovering the error.

**Example 3: Canonical vs. Noindex Conflict**
```html
<meta name="robots" content="noindex, follow">
<link rel="canonical" href="https://site.com/page">
❌ CONFLICT: Can't be both noindexed and canonical
```

**Google's behavior:** Noindex takes priority, canonical ignored.

**Example 4: HTTP vs HTTPS Canonical Mismatch**
```html
<!-- Site URL: https://secure-shop.com/product -->
<link rel="canonical" href="http://secure-shop.com/product">
❌ WRONG: HTTPS page canonicalizing to HTTP version
```

**Impact:** Confuses Google, may index wrong protocol version.

**Canonical Audit Checklist:**

Using Screaming Frog:
```
1. Crawl site
2. URI > Canonical
3. Check for:
   - Missing canonical tags
   - Self-referencing canonical mismatches
   - Canonicals to 404/redirected pages
   - HTTP/HTTPS mismatches
   - Canonical chains
```

**Common Patterns to Fix:**
```
✅ GOOD Canonical Patterns:
- Product variations → Main product
- Paginated pages → Self-referencing OR View All page
- HTTPS pages → HTTPS canonical
- www version → www canonical (or vice versa, be consistent)
- Mobile pages → Desktop version (if separate URLs)

❌ BAD Canonical Patterns:
- Canonical to 404 page
- Canonical chains (A→B→C)
- Different canonical in HTTP headers vs HTML
- Canonical to noindexed page
- Canonical to redirect URL
```

### 3. Cumulative Layout Shift (CLS)

**What It Measures:**

**Good Score:** < 0.1
**Poor Score:** > 0.25


**Visual Example of Layout Shifts:**
```
User tries to click "Read More" button at Y position 450px
↓
Ad loads above content
↓
Button shifts to Y position 950px
↓
User accidentally clicks ad instead ❌
```

**CLS Optimization Techniques:**

**Technique 1: Always Specify Image Dimensions**
```html
<!-- BEFORE: No dimensions specified -->
<img src="article-image.jpg" alt="Article">
<!-- Causes layout shift when image loads -->

<!-- AFTER: Explicit dimensions -->
<img 
  src="article-image.jpg" 
  alt="Article"
  width="800"
  height="600"
  style="max-width: 100%; height: auto;">
<!-- Browser reserves space, no shift -->

<!-- AFTER (Responsive): Use aspect-ratio -->
<img 
  src="article-image.jpg" 
  alt="Article"
  style="width: 100%; aspect-ratio: 4/3;">
```

**Technique 2: Reserve Space for Ads**
```html
<!-- BEFORE: Ad loads and pushes content down -->
<div id="ad-slot"></div>
<script async src="//ads.provider.com/ad.js"></script>

<!-- AFTER: Reserve space with min-height -->
<div id="ad-slot" style="min-height: 250px; background: #f5f5f5;">
  <!-- Ad loads here without shifting content -->
</div>
<script async src="//ads.provider.com/ad.js"></script>

<!-- AFTER (Better): CSS aspect ratio -->
<style>
.ad-container {
  position: relative;
  width: 100%;
  aspect-ratio: 728 / 90; /* Leaderboard ad */
  background: #f5f5f5;
}
.ad-container::before {
  content: 'Advertisement';
  position: absolute;
  top: 5px;
  right: 5px;
  font-size: 10px;
  color: #999;
}
</style>
<div class="ad-container" id="ad-slot"></div>
```

**Technique 3: Font Loading Strategy**


**Microdata (Inline HTML)**

```javascript
      "reviewCount": product.rating.count

  script.textContent = JSON.stringify(schema);
```

```
1. Domain Validated (DV)
   - Validates domain ownership only
   - Cheapest option ($0-$50/year)
   - Issues in minutes
   - Good for: Blogs, small sites

2. Organization Validated (OV)
   - Validates organization details
   - Moderate cost ($50-$200/year)
   - Issues in 1-3 days
   - Good for: Business websites

3. Extended Validation (EV)
   - Extensive validation process
   - Most expensive ($150-$500/year)
   - Issues in 1-2 weeks
   - Good for: E-commerce, financial sites
   - Shows organization name in browser

4. Wildcard Certificate
   - Covers all subdomains
   - Cost: $100-$300/year
   - Example: *.example.com covers:
     - www.example.com
     - shop.example.com
     - blog.example.com
```

**Free SSL Options:**
- Let's Encrypt (automated, 90-day renewal)
- Cloudflare SSL (with Cloudflare CDN)
- AWS Certificate Manager (for AWS hosted sites)

**Step 2: Install Certificate**

**Apache Configuration:**
```apache
<VirtualHost *:443>
    ServerName example.com
    ServerAlias www.example.com
    
    DocumentRoot /var/www/html
    
    SSLEngine on
    SSLCertificateFile /path/to/certificate.crt
    SSLCertificateKeyFile /path/to/private.key
    SSLCertificateChainFile /path/to/ca_bundle.crt
    
    # Modern SSL configuration
    SSLProtocol -all +TLSv1.2 +TLSv1.3
    SSLCipherSuite ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256
    SSLHonorCipherOrder on
    
    # HSTS header
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
</VirtualHost>
```

**Nginx Configuration:**
```nginx
server {
    listen 443 ssl http2;
    server_name example.com www.example.com;
    
    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;
    
    # Modern SSL configuration
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256';
    ssl_prefer_server_ciphers on;
    
    # OCSP stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    
    # HSTS header
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    
    root /var/www/html;
    index index.html;
}
```

**Step 3: Redirect HTTP to HTTPS**

**Apache .htaccess:**
```apache
# Redirect all HTTP to HTTPS
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# Force www version
RewriteCond %{HTTP_HOST} !^www\. [NC]
RewriteRule ^(.*)$ https://www.%{HTTP_HOST}/$1 [L,R=301]
```

**Nginx:**
```nginx
server {
    listen 80;
    server_name example.com www.example.com;
    return 301 https://www.example.com$request_uri;
}
```

### Mixed Content Issues

**What is Mixed Content?**
HTTPS page loading HTTP resources (insecure)

**Real-Life Example:**
```html
<!-- HTTPS page: https://secure.com/page -->
<!DOCTYPE html>
<html>
<head>
  <!-- ❌ Mixed content: HTTP CSS on HTTPS page -->
  <link rel="stylesheet" href="http://cdn.example.com/style.css">
</head>
<body>
  <!-- ❌ Mixed content: HTTP image -->
  <img src="http://images.example.com/logo.png">
  
  <!-- ❌ Mixed content: HTTP script -->
  <script src="http://cdn.jquery.com/jquery.js"></script>
  
  <!-- ❌ Mixed content: HTTP video -->
  <video src="http://videos.example.com/intro.mp4"></video>
</body>
</html>
```

**Browser Behavior:**
- Chrome: Blocks mixed content, shows warning
- Firefox: Blocks mixed content by default
- Safari: Shows warning icon
- Impact: Broken functionality, lost user trust

**Finding Mixed Content:**

**Method 1: Chrome DevTools**
```
1. Open page in Chrome
2. Open DevTools (F12)
3. Console tab
4. Look for warnings:
   "Mixed Content: The page at 'https://...' was loaded over HTTPS,
   but requested an insecure resource 'http://...'"
```

**Method 2: WhyNoPadlock.com**
```
URL: https://www.whynopadlock.com/
Enter HTTPS URL
Shows all mixed content resources
```

**Method 3: Screaming Frog**
```
1. Crawl HTTPS site
2. Security > HTTP (Insecure)
3. Shows all HTTP resources on HTTPS pages
4. Export for bulk fixing
```

**Fixing Mixed Content:**

**Solution 1: Update to HTTPS**
```html
<!-- Change HTTP to HTTPS -->
<link rel="stylesheet" href="https://cdn.example.com/style.css">
<img src="https://images.example.com/logo.png">
<script src="https://cdn.jquery.com/jquery.js"></script>
```

**Solution 2: Protocol-Relative URLs**
```html
<!-- Removes protocol entirely -->
<link rel="stylesheet" href="//cdn.example.com/style.css">
<img src="//images.example.com/logo.png">
<!-- Browser uses same protocol as parent page -->
```

**Solution 3: Content Security Policy**
```html
<meta http-equiv="Content-Security-Policy" content="upgrade-insecure-requests">
<!-- Automatically upgrades HTTP to HTTPS -->
```

**Solution 4: Database Update (WordPress)**
```sql
-- Replace all HTTP with HTTPS in content
UPDATE wp_posts 
SET post_content = REPLACE(
  post_content, 
  'src="http://', 
  'src="https://'
);

UPDATE wp_posts 
SET post_content = REPLACE(
  post_content, 
  'href="http://', 
  'href="https://'
);
```

### Security Headers

**Essential Security Headers for SEO & Trust:**

**1. Strict-Transport-Security (HSTS)**
```
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
```

**What it does:**
- Forces browsers to only use HTTPS
- Prevents SSL stripping attacks
- Once visited, browser remembers to use HTTPS

**Implementation:**
```apache
# Apache
Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
```

```nginx
# Nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
```

**2. Content-Security-Policy**
```
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted-cdn.com; img-src 'self' https: data:; style-src 'self' 'unsafe-inline';
```

**What it does:**
- Prevents XSS attacks
- Controls which resources can load
- Reduces risk of malicious code injection

**Implementation:**
```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; script-src 'self' https://trusted-cdn.com; img-src 'self' https: data:;">
```

**3. X-Content-Type-Options**
```
X-Content-Type-Options: nosniff
```

**What it does:**
- Prevents MIME type sniffing
- Forces browser to respect declared content-type

**4. X-Frame-Options**
```
X-Frame-Options: SAMEORIGIN
```

**What it does:**
- Prevents clickjacking attacks
- Controls if site can be embedded in iframe

**5. Referrer-Policy**
```
Referrer-Policy: strict-origin-when-cross-origin
```

**What it does:**
- Controls referrer information sent to other sites
- Protects user privacy
- Prevents data leakage

**Complete Security Headers Setup:**
```apache
# Apache .htaccess
<IfModule mod_headers.c>
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
    Header always set X-Content-Type-Options "nosniff"
    Header always set X-Frame-Options "SAMEORIGIN"
    Header always set X-XSS-Protection "1; mode=block"
    Header always set Referrer-Policy "strict-origin-when-cross-origin"
    Header always set Permissions-Policy "geolocation=(), microphone=(), camera=()"
    Header always set Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' https://www.google-analytics.com; img-src 'self' https: data:; style-src 'self' 'unsafe-inline';"
</IfModule>
```

```

Enter your URL
Get security rating (A+ to F)
See missing headers and recommendations
```
